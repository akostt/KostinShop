# KostinShop — WPF E-Commerce Management System

## Описание
KostinShop — это настольное приложение на WPF для управления заказами, товарами, категориями, клиентами и ролями пользователей в небольшом интернет-магазине. Приложение поддерживает управление доступом на основе ролей (Администратор, Менеджер, Логист, Клиент), обработку заказов и экспорт в Excel.

## Основные возможности
- **Управление доступом на основе ролей**:
  - **Администратор**: Просмотр таблиц (только чтение); полный контроль над управлением пользователями
  - **Менеджер**: Полный CRUD для товаров, категорий, статусов, клиентов; обработка и редактирование заказов; экспорт заказов в Excel
  - **Логист**: Изменение статуса и адреса доставки заказов
  - **Клиент**: История заказов и каталог товаров

- **Управление заказами**: Создание, редактирование, обработка и отслеживание заказов
- **Каталог товаров**: Управление товарами по категориям с указанием цен
- **Профили клиентов**: Хранение информации о клиентах и баллах лояльности
- **Экспорт в Excel**: Экспорт отдельных заказов в формате .xlsx с цветным оформлением
- **Управление пользователями**: CRUD пользователей с назначением ролей

## Требования к системе
- **.NET SDK**: 9.0 или выше
- **SQL Server**: 2016 или выше (или SQL Express)
- **Visual Studio**: 2022 или выше (опционально; можно собрать через CLI)
- **Windows**: Windows 10 или выше

## Установка и настройка

### 1. Клонирование репозитория
```bash
git clone https://github.com/yourusername/KostinShop.git
cd KostinShop/KostinShop
```

### 2. Конфигурация подключения к базе данных
Отредактируйте `appsettings.json` и установите строку подключения SQL Server:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=ВАШ_СЕРВЕР;Database=KostinShop;Integrated Security=True;TrustServerCertificate=True;"
  }
}
```
Замените `ВАШ_СЕРВЕР` на имя экземпляра SQL Server (например, `(localdb)\\mssqllocaldb` для LocalDB).

### 3. Применение миграций базы данных
```bash
dotnet ef database update
```
Это создает схему базы данных и загружает тестовые данные.

### 4. Сборка приложения
```bash
dotnet build
```

### 5. Запуск приложения
**Вариант A: Через CLI**
```bash
dotnet run
```

**Вариант B: Прямой запуск скомпилированного файла**
```bash
.\bin\Debug\net9.0-windows\KostinShop.exe
```

## Роли пользователей и права доступа

| Роль | Категории | Товары | Клиенты | Статусы | Заказы | Пользователи | Экспорт |
|------|-----------|--------|---------|---------|--------|--------------|---------|
| Администратор | 📖 Чтение | 📖 Чтение | 📖 Чтение | 📖 Чтение | 📖 Чтение | ✏️ Полный | ✗ Нет |
| Менеджер | ✏️ Полный | ✏️ Полный | ✏️ Полный | ✏️ Полный | ✏️ Полный | ✗ Нет | ✅ Да |
| Логист | ✗ Нет | ✗ Нет | ✗ Нет | ✗ Нет | 📍 Статус/Адрес | ✗ Нет | ✗ Нет |
| Клиент | 📖 Чтение | 📖 Чтение | - | - | 📋 Свои | ✗ Нет | ✗ Нет |

Легенда: ✏️ = Полный CRUD, 📖 = Только чтение, 📍 = Частичное редактирование, ✗ = Нет доступа, ✅ = Доступно

## Конфигурация

### Цветовая палитра
Приложение использует пользовательскую цветовую схему, определенную в `Resources/Styles.xaml`:
- **Основной**: RGB(181, 213, 202) — основной цвет интерфейса
- **Дополнительный**: RGB(224, 169, 175) — цвет при наведении
- **Дополнительный 1**: RGB(209, 238, 252) — чередующиеся строки таблиц
- **Дополнительный 2**: RGB(255, 252, 214) — выделение/итоги

### База данных
Приложение использует Entity Framework Core с SQL Server. Миграции находятся в папке `Migrations/`. Для создания новых миграций после изменения моделей:
```bash
dotnet ef migrations add ИмяМиграции
dotnet ef database update
```

## Решение типичных проблем

### "Не удается подключиться к SQL Server"
- Убедитесь, что SQL Server запущен
- Проверьте строку подключения в `appsettings.json`
- Убедитесь, что включена аутентификация Windows или используются учетные данные SQL пользователя

### "Не удается применить миграции"
- Убедитесь, что база данных не существует или используйте `Database.EnsureDeleted()` + `Database.EnsureCreated()` в DbSeeder
- Запустите миграции: `dotnet ef database update`

### "Ошибки компиляции"
- Убедитесь, что установлен .NET 9.0 SDK: `dotnet --version`
- Очистите и пересоберите: `dotnet clean && dotnet build`
